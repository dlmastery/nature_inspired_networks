section 03Deep information · click to expand

Audit methodology — 8 + 8 parallel agents (Tracks A / B / C)

Track A — Implementation Critic Team (8 agents). One agent per hypothesis group (G1..G8) reviewed every src/nature_inspired_networks/*.py module in scope against the design doc. Output: audits/G<X>_audit.md with PASS / MINOR / MAJOR / BROKEN verdicts and an explicit mechanism check (does the code do what the doc claims?).

Track B — Sci-Critic Team (8 parallel agents). A separate cohort performed adversarial research-scientist critique on every hypothesis design doc and appended a verbatim "Addendum: Research-Scientist Critique" section verdicting each as NOVEL / DERIVATIVE / NUMEROLOGY / UNFALSIFIABLE / FALSIFIED — with a +TESTABLE or +CONFIRMED suffix when warranted.

Track C — Fixer Campaign (8 patches). For every MAJOR / BROKEN finding, a Fixer agent (a) patched the implementation, (b) authored a mechanism-verifying unit test, (c) re-ran the affected sweep row at seed 0, and (d) recorded the pre-fix vs post-fix numbers in a FIXER_REPORT.md. The campaign's centerpiece — Fixer-PhiScaling for H09 — uncovered that the stage-param ratio was 1:1.41:2.45 not 1:φ:φ²; the patch corrects to 1:1.623:2.629 (0.43 % max error).

The 8 hypothesis groups (G1–G8) — one-liners
  • G1 — Scaling & Growth (H01–H10). φ / Fibonacci growth laws for depth, width, resolution, parameter budgets, LR schedules.
  • G2 — Layer / Channel / Neuron (H11–H20). Fibonacci MLP sizes, golden skip-modulation, φ-threshold activations, head diversity, ensembles.
  • G3 — Topologies & Graphs (H21–H30). Hexagonal lattices, toroidal closure, Platonic / icosa / dodeca equivariance, cymatic-hex resonance.
  • G4 — Kernels / Attention / Filters (H31–H40). Golden-spiral kernels, Fibonacci dilation, vesica-piscis filters, golden-angle rotary, harmonics.
  • G5 — Optimisation / Init / Regularisation / NAS (H41–H50). Golden-ratio AdamW, φ-weight init, Fibonacci pruning, φ-dropout, golden-momentum schedulers, full-hybrid stacks (the cautionary sg_full_fib is here).
  • G6 — Topological & Bridging (H51–H60). Persistent-homology losses, drop-path anytime nets, icosa-unfold bridges, C4-group avg-pool, trained-feature Betti probes.
  • G7 — Cross-Paradigm Hybrids (H61–H75). Liquid / JEPA / KAN / Transformer / GNN cross-paradigm fusions — no CIFAR sweep rows yet.
  • G8 — Esoteric Extensions (H76–H84). Reuleaux constant-width kernels, SIREN sinusoidal activations, tetrahedral dual paths, radial-12 attention, spectral Hopfield.
Phase 0–8 protocol — what every hypothesis runs through
  • Phase 0 — Unit tests. ideas/<NN>/tests.py green (≥ 4 assertions + one regression test per Rule 12).
  • Phase 1 — CIFAR-10 SOTA-smoke pre-flight. Baseline ResNet-20 must hit ≥ 80 % top-1 at 12 epochs; otherwise STOP and diagnose (Rule 13).
  • Phase 2 — CIFAR-10 hypothesis smoke for ALL hypotheses. 12-epoch ablation per hypothesis; commit + push after each (Rule 11).
  • Phase 3 — Check-in dashboard + RESULTS + FINDINGS refresh. Identify top-K performers by composite. Commit + push.
  • Phase 4 — CIFAR-100 heavy hitters. Only top-K from Phase 3 graduate. ≥ 30 epochs (quality) or 100 + (convergence).
  • Phase 5 — 3-seed re-run on Phase-4 winners. Add error bars before any external claim.
  • Phase 6 — Combo ladder. Orthogonal-axes 2 → N-prior stacks on a verified-winner base (Rule 23).
  • Phase 7 — Dual-track audit. 8 + 8 agent impl-critic + sci-critic pass (Rule 22). This is where the post-fix campaign was triggered.
  • Phase 8 — Fixer + post-fix re-run + reviewer checklist. Per Rule 21, every patched hypothesis is re-smoked before its claim is considered complete.
Reusable skills (7 added during the campaign)

Per Rule 10 every skill is content-agnostic — a future autoresearch project (tabular, medical, FX, …) can pick them up unchanged. The seven added during this audit campaign:

  • autoresearch-multi-agent-dispatch — parallel agents with disjoint file scopes + retry-wrapped commits + index.lock handling.
  • autoresearch-critic-team — implementation audit by group, output audits/G<X>_audit.md with PASS / MINOR / MAJOR / BROKEN verdicts.
  • autoresearch-scicritic-team — research-scientist critique addenda appended directly into design docs.
  • autoresearch-fixer-campaign — patch code, add mechanism- verifying tests, re-run affected sweep rows, record pre-fix vs post-fix table.
  • autoresearch-combo-ladder — orthogonal-axis additive 2 → N-prior stacking on a verified-winner base.
  • autoresearch-per-experiment-page — independent comprehensive dashboard page per run, group-sectioned aggregate, GitHub Pages mirror.
  • autoresearch-auto-checkpoint-loop — background git auto- commit loop for crash safety alongside long-running sweeps and agent teams.
