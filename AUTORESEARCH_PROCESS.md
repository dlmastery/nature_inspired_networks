# AUTORESEARCH_PROCESS

> The 7-step research-loop ritual every experiment in this project follows.
> Inherited from `dlmastery/autoresearchimage` and adapted from medical-OOD
> classification to **ablation studies of nature-inspired priors**.

This document explains the **process**. The full normative rules live in
[`CLAUDE.md`](CLAUDE.md). This is the operator-facing summary.

---

## Why a process?

Ablation studies, especially with many small toggles, are easy to
fudge: cherry-picked configs, post-hoc redefined metrics, optimistic
single-seed numbers. The autoresearch protocol replaces ad-hoc tinkering
with a refusal-to-launch gate stack:

- Every experiment carries an arXiv-cited literature motivation.
- Every experiment writes a numeric prediction **before** training.
- The composite metric is SHA-256 fingerprinted so the formula cannot
  drift between runs.
- The runner appends append-only to `experiment_log.jsonl`; you cannot
  retroactively delete a row without leaving a Git diff.

The process is the deliverable. The accuracy numbers are secondary.

---

## The 7-step ritual

Every experiment follows this exact sequence.

### Step 1 — Diagnose

Read the most recent run from `experiment_log.jsonl`. Identify the
**specific** weakness or open question:

- Which prior contributed the smallest delta?
- Did a leave-one-out reveal a redundancy?
- Did a single-prior-on run *under*-perform vanilla — possibly indicating
  the prior is fighting standard inits?
- What did prior experiments rule out?

Word-count floor: **60 words**; must reference at least one prior
experiment by tag OR a per-row metric.

### Step 2 — Cite

Search arXiv / NeurIPS / ICML / ICLR / CVPR / MICCAI for the **exact**
paper that motivates the change. Format every citation:

```
Author1, Author2, Author3 YEAR VENUE 'Paper Title'
(arXiv:XXXX.XXXXX) — one-sentence note on why we cite it here.
```

The Citation Rigor validator (`src/nature_inspired_networks/reasoning.py::validate_entry`)
rejects:

- `(He2016)` — parenthetical-only tag
- `He et al.` — no year, no venue
- `see README.md` — redirects instead of citing

Word-count floor: **40 words single-paper, 80 words multi-paper**.

### Step 3 — Hypothesize

State the mechanism: **what flag flips, what it does in the model,
what the cited paper predicts**. The hypothesis must contain the
word "mechanism", "because", or "per [paper]".

Word-count floor: **50 words**.

### Step 4 — Predict

Write a numeric outcome range on `composite` **plus** at least one
sub-metric prediction (Δtop-1, Δparams, Δrot-eq-err, Δlatency).
Predictions are stored **before** training in
`reasoning_annotations.json`; the post-run verdict explicitly compares
actual to predicted, so wishful thinking gets caught.

Word-count floor: **25 words**; must include a numeric range.

### Step 5 — Execute

ONE config change per experiment. In this project that means flipping
exactly one flag in `flags:` or switching the `channel_mode`.
The runner enforces this by tag convention:

- `baseline_*` — reference points (resnet20, sg_vanilla)
- `sg_chan_<mode>` — channel-scaling-only
- `sg_only_<flag>` — single-prior-on
- `sg_loo_no_<flag>` — leave-one-out from full
- `sg_full_<mode>` — all priors on with chosen channel mode

The runner also writes the Goodhart-fingerprint check (`COMPOSITE_FINGERPRINT`
in `src/nature_inspired_networks/eval.py`) to each `metrics.json` and refuses to publish
if the running fingerprint differs from the first-run fingerprint.

### Step 6 — Analyze

Compare actual to predicted:

- Did the prediction hold? Why / why not?
- Update the **verdict** field with `KEEP` / `DISCARD` / `NEAR-MISS`,
  the exact composite to 4 decimals, the delta vs. the global best,
  and per-prior narrative.

Word-count floor: **30 words**; must include status label + composite +
at least one per-prior delta.

### Step 7 — Checkpoint

Update every artifact in the **Dashboard Files Update Mandate**:

- `experiments/<dataset>/<tag>_seed<S>/metrics.json` — written by runner
- `experiments/<dataset>/<tag>_seed<S>/history.json` — written by runner
- `experiments/experiment_log.jsonl` — runner appends one row
- `reasoning_annotations.json` — operator writes pre-run + post-run
- `memory/project_checkpoint.md` — operator updates with
  next-experiment command
- `dashboard/dashboard.html` — regenerated by `scripts/build_dashboard.py`

Word-count floor on **learning**: **40 words**; must include
"axis closed" / "axis open" language OR a concrete "next try: …".

## How the gates compose

```
                         ┌─────────────────────────────┐
                         │ Pre-run reasoning entry      │
                         │   (steps 1-4 above)          │
                         └────────────┬────────────────┘
                                      │
                                      ▼
              ┌─────────────────────────────────────────┐
              │ Citation Rigor validator                │
              │   author + year + venue + title +       │
              │   arXiv ID + relevance note per paper   │
              └────────────┬────────────────────────────┘
                           │ (else: refuse)
                           ▼
              ┌─────────────────────────────────────────┐
              │ Reasoning Blob Completeness validator   │
              │   word-count floors per field           │
              └────────────┬────────────────────────────┘
                           │ (else: refuse)
                           ▼
              ┌─────────────────────────────────────────┐
              │ Goodhart-fingerprint check on composite │
              │   SHA-256(formula) must match the one   │
              │   stored at first run                   │
              └────────────┬────────────────────────────┘
                           │ (else: refuse)
                           ▼
                    ╔═══════════════╗
                    ║  TRAIN + EVAL ║
                    ╚═══════════════╝
                           │
                           ▼
              ┌─────────────────────────────────────────┐
              │ Per-experiment artifact write           │
              │   (Dashboard Files Update Mandate)      │
              └─────────────────────────────────────────┘
```

There is **no `--bypass` flag**.

## Composite metric — Goodhart fingerprint

```python
composite = top1 − 0.05·log₁₀(params_M) − 0.05·log₁₀(latency_ms)
```

- `top1` ∈ [0, 1]
- `params_M = max(0.001, params / 1e6)`
- `latency_ms = max(0.01, gpu_latency_ms)` (batch=1)

The string above is SHA-256 hashed at import (`COMPOSITE_FINGERPRINT`
in `src/nature_inspired_networks/eval.py`). Every `metrics.json` records that hash; any
mid-project edit raises a runtime error on the next sweep.

Rationale: ablation studies are uniquely vulnerable to composite-metric
drift — a researcher staring at disappointing accuracy numbers is
tempted to add a "params penalty term" that retroactively crowns a
favoured row. The fingerprint forecloses that.

## Empirical evidence the gates fire

The gates only earn their keep when they reject something. In this
project so far:

1. **`HexConv2d` masked-weight forward initially silently allowed
   gradient leakage to corner taps.** The first ablation row caught
   it: `sg_only_hex` showed *no* parameter delta vs. vanilla because
   `nn.Conv2d` carried gradient through the masked positions despite
   the inference-time mask. Fix: apply the mask **inside** `forward`
   so backward respects it (`priors.HexConv2d.forward`).

2. **C4 `GroupConv2d` produced wrong shape at stride 2 in the initial
   draft** because group-axis reduction happened before the spatial
   downsample. Caught by the model smoke test (`(4, 10)` shape assert).
   Fix: stride is applied inside the per-orbit `F.conv2d` call, then
   the group axis is max-pooled.

3. **CIFAR download blocked by `Python 3.13 SSL: Basic Constraints of
   CA cert not marked critical`** on the corporate cert chain. Caught
   by the very first dataset call. Fix: out-of-band download by
   `curl.exe -k`, verified by torchvision's md5 check.

These are the kinds of bugs that make ad-hoc ablation studies useless.
The gates surface them before they corrupt a number that would later
end up in a paper.

## Reading a row from `experiment_log.jsonl`

Each line is one experiment. Schema:

```json
{
  "tag":               "sg_full_fib",
  "dataset":           "cifar10",
  "seed":              0,
  "epochs":            12,
  "top1":              0.8643,
  "top5":              0.9961,
  "params":            259200,
  "flops":             216608544.0,
  "latency_ms":        14.32,
  "rot_eq_err":        0.281,
  "composite":         0.8378,
  "composite_fingerprint": "d65565e9c7b1...0d0893",
  "epochs_to_target":  9,
  "train_seconds":     410.3,
  "train_top1":        0.9012,
  "generalization_gap":0.0369,
  "model":             "NaturePrior",
  "channel_mode":      "fib",
  "flags": {"hex": true, "group": true, ...},
  "composite_formula": "composite = top1 - 0.05 * log10(params_M) - 0.05 * log10(latency_ms)"
}
```

Append-only; the dashboard regenerates the table view, never the data.
